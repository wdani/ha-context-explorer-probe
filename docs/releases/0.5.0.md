# HA Context Explorer 0.5.0

Tag: `0.5.0`

## Summary

`0.5.0` is the first distribution-ready custom repository milestone after the internal Home Assistant domain rename to `ha_context_explorer`.

This release prepares HA Context Explorer for a cleaner manual/HACS custom repository install path while keeping the integration strict read-only, local-first, and privacy-aware.

## Highlights

- Final integration domain is now `ha_context_explorer`.
- Current runtime paths no longer use the old probe-era domain.
- API endpoints are under `/api/ha_context_explorer/...`.
- Sidebar panel path is `/ha_context_explorer`.
- Static frontend path is `/local/ha_context_explorer`.
- Native panel lifecycle/interactivity handling was hardened during the rename work.
- Developer Workbench foundation is included for admin-only local diagnostics and review exports.
- HACS custom repository metadata exists.
- Local integration brand assets exist under `custom_components/ha_context_explorer/brand/`.
- Home Assistant Brands helper assets are prepared under `docs/brands/home-assistant-brands/custom_integrations/ha_context_explorer/`.

## Breaking Change

The old probe-era integration domain `ha_context_explorer_probe` is not automatically migrated.

Existing old-domain installs may need manual cleanup before installing `0.5.0`.

## Manual Migration From Old Probe Domain

1. Remove the old HA Context Explorer integration entry if present.
2. Remove the old `custom_components/ha_context_explorer_probe` folder.
3. Install or copy the new `custom_components/ha_context_explorer` folder.
4. Restart Home Assistant.
5. Add HA Context Explorer again from Settings -> Devices & Services.
6. Clear browser cache if old assets or the old panel path remain visible.

## Safety And Trust Model

HA Context Explorer remains strict read-only:

- GET-only real data endpoints
- authenticated/admin-only access for real Home Assistant data
- no service calls
- no mutation endpoints
- no Home Assistant config writes
- no `.storage` reads or writes
- no secret access
- no telemetry or remote diagnostic upload
- local-only Developer Workbench exports
